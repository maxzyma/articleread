# 翻译质量检查清单

本文档提供系统化的翻译质量检查维度，用于批判性审查机器翻译输出。

---

## 检查维度概述

### 1. 严重误译（Critical Issues）
**定义**：导致读者无法理解原文意思，或产生完全错误理解的翻译

**检查项**：
- [ ] 比喻/习惯用语是否失效
  - 例如："草稿本数学" → 应为"草稿本上的计算过程"
  - 例如："发光" → 应为"发挥优势"（中文语境不当）
- [ ] 关键名词误译
  - 例如："卧室"（room 的空间含义）误译为建筑构件
  - 例如："螺栓连接"（bolted）应为"生硬嫁接"
- [ ] 动词/动作完全错误
  - 例如："回报"（report back）误译为"回报"（reward）

**修复优先级**：⚠️ **必须修复** - 直接影响理解

---

### 2. 表达不当（Major Issues）
**定义**：可以理解，但读起来生硬、不符合中文习惯的表达

**检查项**：
- [ ] 典型的机器直译模式
  - "戏剧性地更容易" → 应为"显著简化"
  - "通过...反弹" → 应为"通过...中转"
- [ ] 词序不符合中文习惯
  - "在 TAUBench 上提升 5% 性能" → "在 TAUBench 上性能提升 5%"
- [ ] 用词不够精准
  - "完全智能体" → "全功能智能体"
  - "进入"（发布 API）→ "推出"

**修复优先级**：⚠️ **建议修复** - 影响阅读体验

---

### 3. 术语不一致（Minor Issues）
**定义**：技术术语翻译不统一，或不符合行业标准

**检查项**：
- [ ] 同一概念多次出现，翻译不统一
- [ ] 技术术语首次出现未注明英文
- [ ] 行业标准术语未采用
  - 例如："智能体"（Agent）而非"代理人"
  - 例如："推理模型"（Reasoning Models）而非"思考模型"

**修复优先级**：ℹ️ **建议优化** - 影响专业性

---

### 4. 语境错误（Contextual Issues）
**定义**：未理解原文语境导致的误译

**检查项**：
- [ ] 专业术语在上下文中的具体含义
  - 例如："state" 在 API 语境中是"状态"，而非"州"
- [ ] 比喻性表达在特定领域中的含义
  - 例如："侦探调查"比喻用于解释"智能体循环"
- [ ] 行业惯用语
  - 例如："胶带"（duct tape）在技术语境中指"临时拼凑的方案"

**修复优先级**：⚠️ **建议修复** - 可能导致误解

---

## 检查流程

### 阶段 1：快速扫描（识别严重问题）
1. 通读全文，标记无法理解的句子
2. 检查关键比喻和习惯用语
3. 验证核心概念的翻译准确性

### 阶段 2：逐句审查（识别表达问题）
1. 逐句阅读，标记生硬表达
2. 检查技术术语一致性
3. 验证专业术语的准确性

### 阶段 3：术语验证（统一性检查）
1. 建立术语对照表
2. 全文搜索同一概念的不同翻译
3. 统一为行业标准或首次使用的翻译

## 常见错误模式

### 错误模式 1：比喻失效
**特征**：直译比喻，导致中文无法理解
**示例**：
- "草稿本数学" → "草稿本上的计算过程"
- "发光" → "发挥优势" / "大显身手"

### 错误模式 2：动词误选
**特征**：多义词选择错误
**示例**：
- "report" → "汇报"（而非"报告"在某些语境）
- "return" → "返回"（而非"回报"）

### 错误模式 3：修饰语直译
**特征**：副词/形容词生硬直译
**示例**：
- "dramatically easier" → "显著简化"（而非"戏剧性地更容易"）
- "famously built" → "著名的周末速成"（而非"著名地建造"）

### 错误模式 4：多义词未按语境选择
**特征**：常见词在专业语境中的特殊含义未正确翻译
**示例**：
- "room"（空间）→ "空间"（而非"卧室"）
- "tape"（临时修补）→ "临时拼凑"（而非"胶带"）
- "bolted"（强行添加）→ "生硬嫁接"（而非"螺栓连接"）
