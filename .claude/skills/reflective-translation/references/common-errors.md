# 常见机器翻译错误模式

本文档总结机器翻译（特别是 LLM 翻译）的常见错误模式及修复方法。

---

## 错误模式分类

### 1. 习惯用语和比喻失效（Idiom & Metaphor Failures）

#### 问题特征
- 直译英语习语，导致中文完全无法理解
- 比喻性表达在目标语言中失效

#### 常见案例

| 原文 | 错误翻译 | 正确翻译 | 原因 |
|------|---------|---------|------|
| "shine" | "发光" | "发挥优势" / "大显身手" | 语境不当 |
| "notebook math" | "草稿本数学" | "草稿本上的计算过程" | 直译 |
| "duct tape" | "胶带" | "临时拼凑的方案" | 文化差异 |
| "silver bullet" | "银子弹" | "万能解决方案" | 习语未本地化 |

#### 检查技巧
1. 标记文中所有比喻性表达
2. 验证比喻在中文语境中是否自然
3. 必要时改用明喻或解释性翻译

---

### 2. 多义词选择错误（Polysemy Errors）

#### 问题特征
- 多义词未根据语境选择正确含义
- 选择最常见的字典释义，忽略专业含义

#### 常见案例

| 原文 | 语境 | 错误翻译 | 正确翻译 |
|------|------|---------|---------|
| report | 侦探调查 | "报告" | "汇报" |
| return | 函数返回 | "回报" | "返回" |
| room | API 设计 | "卧室" | "空间" |
| state | 状态管理 | "州" | "状态" |
| monitor | 系统监控 | "班长" | "监控" |

#### 检查技巧
1. 识别多义词的上下文
2. 查阅专业术语表
3. 验证所选含义是否符合整体逻辑

---

### 3. 修饰语生硬直译（Awkward Modifiers）

#### 问题特征
- 副词和形容词逐字翻译，导致表达生硬
- 英语修饰语与中文搭配不当

#### 常见案例

| 原文 | 错误翻译 | 正确翻译 |
|------|---------|---------|
| dramatically easier | "戏剧性地更容易" | "显著简化" |
| famously built | "著名地建造" | "著名的速成" |
| fully integrated | "完全集成" | "全面集成" |
| fundamentally multimodal | "根本性地多模态" | "从根本上多模态" |

#### 修复模式
1. **-ly 副词**：
   - dramatically → 显著/极大地
   - fundamentally → 根本上/本质上
   - substantially → 实质性地/大幅地

2. **-ful 形容词**：
   - powerful → 强大/有力
   - helpful → 有帮助/有益
   - meaningful → 有意义/重要

---

### 4. 动词-名词搭配不当（Collocation Errors）

#### 问题特征
- 动词和名词的搭配不符合中文习惯
- 直接使用英语的动宾搭配

#### 常见案例

| 原文 | 错误翻译 | 正确翻译 |
|------|---------|---------|
| bounce off backend | "通过后端反弹" | "通过后端中转" |
| enter API | "进入 API" | "推出 API" |
| give background | "给出背景" | "提供背景" |
| take actions | "采取行动" | "执行操作" |

#### 检查技巧
1. 检查动词-名词搭配是否符合中文习惯
2. 查阅该领域的平行文本
3. 必要时更换动词或调整表达

---

### 5. 技术术语不一致（Terminology Inconsistency）

#### 问题特征
- 同一技术概念在不同位置使用不同翻译
- 未遵循行业标准的术语翻译

#### 常见不一致案例

| 英文 | 不一致翻译 | 统一翻译 |
|------|----------|---------|
| Agent | 代理人/智能体/行动者 | 智能体 |
| Reasoning Model | 推理模型/思考模型/理性模型 | 推理模型 |
| API | 接口/API | API（保留原文） |
| Multimodal | 多模态/多模式 | 多模态 |

#### 检查技巧
1. 首次出现时建立术语表
2. 全文搜索同一概念的不同翻译
3. 参考行业权威翻译

---

### 6. 逻辑关系错误（Logical Relationship Errors）

#### 问题特征
- 连词、介词误译导致逻辑关系改变
- 否定、条件关系理解错误

#### 常见案例

| 原文 | 错误翻译 | 正确翻译 |
|------|---------|---------|
| not only A but also B | 不仅 A，而且 B | 不仅 A，而且 B |
| either A or B | 或者 A 或者 B | 要么 A 要么 B |
| while A, B | 当 A 时，B | 虽然 A，但是 B |
| unless A | 除非 A | 如果不 A |

---

### 7. 文化和行业语境缺失（Cultural/Domain Context Gaps）

#### 问题特征
- 未理解特定行业或文化背景
- 直译导致专业含义丢失

#### 常见案例

| 原文 | 语境 | 错误翻译 | 正确翻译 |
|------|------|---------|---------|
| weekend project | 开发者文化 | "周末项目" | "周末速成" |
| duct tape solution | 工程文化 | "胶带方案" | "临时拼凑" |
| silver bullet | 软件工程 | "银子弹" | "万能方案" |
| technical debt | 软件工程 | "技术债" | "技术债务" |

---

## 错误模式检测算法

### 自动检测思路

1. **比喻检测**：
   ```python
   metaphor_keywords = ["像", "如同", "比喻"]
   literal_translation = check_if_metaphor_fails()
   ```

2. **多义词检测**：
   ```python
   polysemous_words = load_common_polysemous_words()
   context = analyze_context(word)
   translation = select_meaning_by_context(word, context)
   ```

3. **搭配检测**：
   ```python
   collocation = verb + noun
   if collocation not in common_chinese_corpus:
       flag_for_review()
   ```

## 修复优先级

### P0 - 必须立即修复（阻塞理解）
- 比喻完全失效
- 核心概念误译
- 关键动词错误

### P1 - 应该尽快修复（影响体验）
- 生硬的表达
- 不自然的搭配
- 逻辑关系错误

### P2 - 可以延后修复（影响专业度）
- 术语不一致
- 轻微的语序问题
- 个别词汇不够精准

---

## 预防策略

### 翻译前
1. 明确文档类型（技术文档/博客/新闻）
2. 确定目标读者（开发者/普通用户/专业人士）
3. 收集领域术语表

### 翻译中
1. 对比喻性表达添加注释
2. 对多义词标注上下文
3. 对首次出现的术语保留英文

### 翻译后
1. 运行术语一致性检查
2. 通读全文检查流畅度
3. 请领域专家审阅关键段落
