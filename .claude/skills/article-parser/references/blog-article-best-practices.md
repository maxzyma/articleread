# 博客/个人站点全文提取最佳实践

## 特点

博客和个人站点是最友好的提取来源，通常无限制。

---

## 支持的平台

| 平台 | 域名示例 |
|------|----------|
| Medium | `medium.com`, `*.medium.com` |
| Dev.to | `dev.to` |
| 简书 | `jianshu.com` |
| 个人博客 | 各类域名 |
| GitHub Pages | `*.github.io` |
| Substack | `*.substack.com` |

---

## 提取流程

```bash
# 访问文章
navigate_page -> <博客 URL>

# 获取页面快照
take_snapshot
```

---

## Medium 特殊处理

Medium 文章可能有阅读限制：
- 会员专属文章需要登录
- 每月免费文章数量有限

**处理方式**：
1. 尝试直接访问
2. 如遇付费墙，搜索其他来源

---

## 图片处理

| 平台 | 图片策略 |
|------|----------|
| Medium | 可直接引用 `miro.medium.com` |
| Dev.to | 可直接引用 |
| 个人博客 | 检查是否有防盗链 |
| GitHub | 可直接引用 |

---

## 代码块处理

博客通常包含代码示例：
- 保留完整代码块
- 保留语法高亮标记
- 注意代码中的特殊字符转义

```markdown
```python
def example():
    pass
```
```
