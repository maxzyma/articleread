## 为什么选择 Claude 做 Agent？

### 1. **Chatbot vs Agent 的本质差异**

张涛指出，过去两年所有大语言模型的训练目标（Pre-training 和 Post-training）都是面向 Chatbot 场景：

| 特性 | Chatbot 模式 | Agent 模式 |
|------|------------|-----------|
| **训练目标** | 在一轮对话中回答问题 | 完成多步骤任务 |
| **行为模式** | 快速给出答案 | 深度思考、逐步执行 |
| **用户期望** | 立即反馈 | 任务完成 |

**核心问题**：绝大部分模型的对齐目标是做 Chatbot，因此用这些模型做 Agent，会发现它们特别希望快速给出答案，而不是逐步解决问题。

### 2. **Claude 的差异化优势**

与其他模型相比，Claude 在 Agent 场景的优势：

✅ **投入更早**：Claude 在 Agent 场景的投入比其他模型更早
✅ **投入更深**：从 Pre-training 到 Post-training 全流程针对 Agent 场景优化
✅ **多步推理**：不会急于在一轮对话中给出答案，能够处理复杂的多步骤任务
✅ **最佳实践**：Manus 研发过程中测试最多的模型是 Claude 3.7 Sonnet

### 3. **市场验证**

张涛明确表示：
> "目前在市面上来说，如果要做 Agent，现在最佳的模型还是 **Claude 3.7 Sonnet**，这也是我们研发过程中最多的选择。"

## Manus 产品核心理念

### "Less Structure, More Intelligence"

与其他 Agent 产品不同，Manus 坚持一个重要的基础理念：

❌ **不做复杂预设**：没有使用预先设置的 workflow 等复杂能力
✅ **让智能涌现**：所有能力都是从 Agent framework 中涌现出来的
✅ **更多上下文**：给模型提供更多上下文，减少对模型的控制
✅ **释放智力**：让模型的智力能够更大发挥出来

### 三个关键点

1. **虚拟机隔离**：给每个 Manus 任务分配一台虚拟机，让它可以使用各种工具
2. **权限与资源**：提前采买私有化数据库和各种 API
3. **训练与磨合**：如果结果不合要求，让 Manus 先确认清楚细节再开始工作（像培养实习生）

## 技术启示

对于正在构建 AI Agent 应用的开发者：

1. **模型选择**：优先考虑 Claude 3.7 Sonnet，特别是 Agent 场景
2. **设计理念**：减少预设结构，给模型更多上下文和自主空间
3. **任务设计**：Agent 任务应支持多步骤执行，而非追求快速答案
4. **用户体验**：像培养实习生一样，允许磨合和确认细节的过程
